# Customer-Purchase-Behaviour-Analysis
A complete SQL-based analytics project exploring customer purchase patterns, product performance, and revenue trends for a retail business. This project demonstrates practical data analysis skills using SQL Server, including database design, data cleaning, aggregations, joins, CTEs, and RFM (Recency, Frequency, Monetary) analysis.
# Project Overview
This project simulates a retail environment with customers, products, and transactions.
Using SQL Server, the goal is to analyze customer behavior and generate insights that help businesses improve retention, product strategy, and revenue growth.

The project includes:

Database schema creation

Sample data insertion

Analytical SQL queries

RFM scoring

Customer segmentation

Revenue and product insights
# Database Structure
The project uses three core tables:

1. Customers
Contains customer demographic and onboarding details.

2. Products
Contains product information including category and price.

3. Transactions
Captures purchase activity including quantity and date.
# Files Included
schema.sql — Creates database and tables
data.sql — Inserts sample data
analysis_queries.sql — Contains all analytical SQL queries
# Key Analyses Performed
1. Total Revenue Calculation
Compute total revenue generated from all transactions.

2. Top Customers by Spending
Identify high-value customers based on total purchase amount.

3. Most Sold Products
Find products with the highest sales volume.

4. Category-Level Revenue
Analyze revenue contribution by product category.

5. Monthly Revenue Trend
Track sales performance over time.

6. RFM Analysis (Recency, Frequency, Monetary)
Segment customers based on:

Recency — How recently they purchased

Frequency — How often they purchased

Monetary — How much they spent

7. Repeat Customer Identification
Find customers who made multiple purchases.

8. Customer Retention Rate
Calculate retention percentage based on transaction activity.
